hts_multistream

Usage
usage: hts_multistream.py [-h] [--clk CLK] [--trg TRG] [--sim SIM] [--trace TRACE]
                          [--auto_soft_trigger AUTO_SOFT_TRIGGER] [--clear_counters] [--spad SPAD]
                          [--decimate DECIMATE] [--nbuffers NBUFFERS] [--secs SECS] [--map MAP]
                          [--sig_gen SIG_GEN] [--delete DELETE] [--recycle RECYCLE] [--check CHECK]
                          [--dry_run DRY_RUN] [--wrtd_txi WRTD_TXI] [--SIG_SRC_TRG_0 SIG_SRC_TRG_0]
                          [--SIG_SRC_TRG_1 SIG_SRC_TRG_1] [--RTM_TRANSLEN RTM_TRANSLEN]
                          [--mtrg MTRG] [--verbose VERBOSE]
                          uutnames [uutnames ...]

Positional Arguments

uutnames

uuts

Named Arguments

--clk

int|ext|zclk|xclk,fpclk,SR,[FIN]

--trg

int|ext,rising|falling

--sim

s1[,s2,s3..] list of sites to run in simulate mode

--trace

1 : enable command tracing

--auto_soft_trigger

force soft trigger generation

Default: 0

--clear_counters

clear all counters SLOW

Default: False

--spad

scratchpad, eg 1,16,0

--decimate

decimate amount

--nbuffers

max capture in buffers

Default: 5000

--secs

max capture in seconds

Default: 0

--map

uut:port:site ie –map=67:A:1/67:B:2/130:BOTH:ALL

Default: “ALL:BOTH:ALL”

--sig_gen

Signal gen to trigger when all uuts armed

--delete

delete stale data

Default: 1

--recycle

overwrite data

Default: 1

--check

run tests simulate ramp=1 or spad sequential=2

Default: 0

--dry_run

run setup but dont start streams or uuts

Default: 0

--wrtd_txi

Command first box to send this trigger when all units are in ARM state

--SIG_SRC_TRG_0

Set trigger d0 source

--SIG_SRC_TRG_1

Set trigger d1 source

--RTM_TRANSLEN

Set rtm_translen for each uut

--mtrg

value:HDMI|EXT, works with free-running master trigger

--verbose

increase verbosity

Default: 0

Outline:

hts_multistream High Throughput Stream from up to 16 UUTS

  • data on local SFP/AFHBA

  • control on Ethernet

DEPRECATION WARNING: please consider using AFHBA404/HAPI/hts_multistream.py

example usage:

./user_apps/acq2106/hts_multistream.py acq2106_133 acq2106_176

./user_apps/acq2106/hts_multistream.py --nbuffers=2000 acq2106_133 acq2106_176

./user_apps/acq2106/hts_multistream.py ---map=133:A:1/133:B:2 --secs=30 acq2106_133

./user_apps/acq2106/hts_multistream.py --map=643:A:ALL/124:BOTH:3,4 --nbuffers=1000 acq2106_067

./user_apps/acq2106/hts_multistream.py --spad=1,8,1 --map=11:C:2 --secs=3600 --check=2 z7io_011

Recommendation: –secs is really a timeout, use –nbuffers for exact data length

Warning: If data rate exceeds bandwidth uut will stay in arm
